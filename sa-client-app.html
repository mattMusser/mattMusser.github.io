Shoff Accounting Client App

About

Shoff Accounting Client App is a Ruby on Rails application that allows the admin to add/remove clients to the app, add links of financial review spreadsheets to the respective client's profiles, and gives the clients of Shoff Accounting easy access to their financial reports. Users are able to up/down vote posts as well as favorite them.

Explanation

The owner of Shoff Accounting, Jonathan, needed an app in which he could give his clients easy access to their financial reports instead of them having having to go back and search through their email inbox for the emails he sends them containing the link(s) to the reports. He also needed client managment tools so he or an admin could add or remove clients to and from the app and add links, to google spreadsheets containing financial reports, to a client's account.

Starting Process

  • Step 1

    • I started of this project by creating a new Rails app by running rails new shoff-accounting --skip-test-unit in the command line and generating the basic files and file structure.

  • Step 2

    • Once that was complete I went to the Gemfile added a few gems that I believe that would need for this project such as Rails 5.1.4, Boostrap, Haml, Devise, Bcrypt, Pundit, and Faker.

  • Step 3

    • I knew I would need a welcome controller, a users controller, and a link controller so I went ahead and generated them. After generating the controllers I proceeded to generate the models for users and links. Since he wanted to add links to his users account I knew I needed to create a seperate controller and model for them so I would be able to add the functionality he wanted.

Problem

The major problem I ran into with this project was the the admin dashboard I wanted to implement to give him the client managment tools he needed. I first started by trying to create the admin dashboard from scratch but I wasn't able to get the functionality I needed and get it looking polished.

Solution

Seeking the advise of a fellow developer he pointed me to a gem called ActiveAdmin which provides an admin dashboard framework so I wouldn't have to struggle building one from scratch.

To implment the tools to add/remove clients and to add links I had to create a file for users and links in the active admin directory to create the views under ActiveAdmin. For the user dashboard view I created at table that provided him with information of each client and a link to remove the client on each row. Then to give him the ability to add clients I added a button in the top right of the dashboard that would send him to a new user form.

For the Links view I also created a table that showed each Client he had and the links associated with their account. For each user's row he I gave him links links that would let him edit or remove a link from a client's account. Just as I did with the Users view I added a button that would send him to a page with a form he could select a client based on their client id and add a link to their account.

Problem

After creating the users and links views for the admin dashboard I ran into an issue where I wasn't able to create client, edit a client's information, or add a link to a client's account.

Solution

To solve this I turned to the docs for ActiveAdmin. In order to be able to change those attributes of a client I had to define which attributes could be changed by using permit_params. I was able to fix the user creation/edit issue by permiting the dashboard access to the :client_name, :email, :password, :password_confirmation attributes. To get the Link functionality working I had to permit the dashboard access to permit_params :url, :user_id attributes.
